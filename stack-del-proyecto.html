Tecnologías en detalle

🐍

Python

Backend

El lenguaje principal. Todo el procesamiento de datos, las consultas a la base de datos y la lógica de negocio está escrita en Python.

⚗️

Flask

Backend — Framework Web

Micro-framework que convierte Python en un servidor web. Define las rutas / y /admin y conecta todo.

🗄️

SQLite

Base de Datos

Base de datos relacional en un solo archivo bienes_raiz.db. Guarda lotes, casas y carros en la tabla propiedades.

🧩

Jinja2

Frontend — Templates

Motor de plantillas de Flask. Permite usar variables de Python dentro del HTML con la sintaxis {{ variable }} y {% if %}.

🎨

Bootstrap 5.3

Frontend — CSS Framework

Librería de estilos. Se usa para la barra de navegación, la cuadrícula de tarjetas y los botones. Cargado desde CDN.

✏️

CSS Personalizado

Frontend — Estilos Propios

Archivo static/css/estilo.css con los estilos únicos del sitio: colores dorados, tarjetas de categoría, botón de WhatsApp.

🦄

Gunicorn

Despliegue — Servidor WSGI

Servidor de producción que reemplaza al servidor de desarrollo de Flask. Es obligatorio en Render para manejar múltiples usuarios.

☁️

Render

Despliegue — Plataforma Cloud

Hosting gratuito donde corre el sitio en producción. Configurado con el archivo render.yaml.

🐙

Git + GitHub

Herramienta — Control de Versiones

Guarda el historial de cambios del código. Render se conecta a GitHub para hacer deploy automático.

🖼️

Imágenes WebP

Frontend — Assets

Formato moderno de imagen más ligero que JPG/PNG. Guardadas en static/img/ para cargas más rápidas.

🧠 Lo que aprendemos de este stack — Metodología

1

ENTENDER: Este es un sitio web dinámico de 2 páginas: la vitrina pública (/) y el panel de administración (/admin). El usuario ve propiedades, las filtra, y el dueño las agrega desde el panel.

2

PENSAR: El flujo es: navegador → Flask recibe la petición → consulta SQLite → Jinja2 inyecta los datos en el HTML → el usuario ve la página con Bootstrap bien estilada.

3

ESCRIBIR: Cada pieza tiene un rol claro. Flask es el "cerebro", SQLite es la "memoria", Jinja2 es el "pintor" que mezcla datos con HTML, Bootstrap es el "sastre" que da el estilo.

4

VERIFICAR: El stack es correcto para un MVP pequeño. La limitación es SQLite — en producción con muchos usuarios simultáneos, habría que migrar a PostgreSQL. Render ya lo soporta.
